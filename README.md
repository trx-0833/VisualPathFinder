# VisualPathFinder - 可视化路径搜索算法演示工具

[![Python](https://img.shields.io/badge/Python-3.8%2B-blue.svg)](https://python.org)
[![Pygame](https://img.shields.io/badge/Pygame-2.5.0-green.svg)](https://pygame.org)
[![OpenCV](https://img.shields.io/badge/OpenCV-4.8.0.74-orange.svg)](https://opencv.org)
[![License](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

一个基于 Pygame 和 Tkinter 的交互式路径搜索算法可视化工具，支持多种经典搜索算法，并包含迷宫生成、图像导入和实时可视化功能。

## ✨ 功能特性

### 🧠 算法可视化
- **广度优先搜索 (BFS)** - 找到最短步数路径
- **深度优先搜索 (DFS)** - 快速找到一条路径
- **Dijkstra算法** - 最小代价路径（考虑权重）
- **A*算法** - 启发式搜索，高效找到最优路径
- **双向广度优先搜索 (D-BFS)** - 从起点和终点同时进行BFS搜索
- **双向深度优先搜索 (D-DFS)** - 从起点和终点同时进行DFS搜索
- **B*算法** - 结合最佳优先搜索和A*的改进算法
- **实时过程显示** - 绿色节点表示已访问，紫色节点表示待访问

### 🗺️ 迷宫系统
- **随机迷宫生成** - 支持设置随机种子复现迷宫
- **图片迷宫导入** - 可将图片转换为迷宫（黑白二值化处理）
- **多种地形类型** - 障碍物（黑色）、普通路径（白色）、高成本区域（灰色/橙色）
- **权重系统** - 不同颜色代表不同的通行成本

### 🎮 交互功能
- **点击设置起点/终点** - 直观的鼠标操作
- **实时路径显示** - 算法执行过程中动态显示搜索过程
- **性能分析** - 集成cProfile，显示算法运行统计
- **可调参数** - 网格大小、搜索细节显示、随机种子等

## 🚀 使用指南

### 基本使用
1. **启动程序**
   ```bash
   python main.py
   ```

2. **界面介绍**
   - 左侧：迷宫显示区域（Pygame）
   - 右侧：控制面板（Tkinter）

3. **设置起点和终点**
   - 方法1：点击起点/终点输入框，然后在迷宫图中点击设置位置
   - 方法2：直接在输入框中输入坐标，格式如 `(5, 10)`

4. **选择搜索算法**
   - 使用下拉菜单选择BFS、DFS、Dijkstra、A*、D-BFS、D-DFS或B*算法
   - 勾选"显示搜索细节"可观察算法展开过程

5. **控制面板功能**
   - **随机种子**：输入种子值并点击"设置种子"按钮，然后点击"刷新地图"使用指定种子
   - **重置**：清除当前显示的路径
   - **确定**：确认输入框中的起点和终点位置
   - **刷新地图**：生成新的随机迷宫
   - **搜索路线**：执行选定的路径搜索算法
   - **显示搜索细节**：实时显示算法的搜索过程

### 高级功能

#### 图片迷宫导入
1. 点击"图片迷宫探索(实验性)"按钮
2. 选择一张图片文件（推荐JPG格式，路径不要包含中文）
3. 程序将自动将图片转换为黑白迷宫
4. **注意**：建议将GRID_NUM设置为LENGTH/2以获得最佳效果

#### 算法性能分析
- 每次搜索路径时，程序会自动运行cProfile进行性能分析
- 分析结果会显示在控制台，包括函数调用次数和耗时统计

## 🔧 故障排除

### 常见问题

1. **程序无法启动**
   ```
   确保安装了正确版本的Pygame和OpenCV
   ```

2. **图片导入失败**
   ```
   确保图片路径不包含中文字符
   检查图片格式是否支持（推荐JPG）
   尝试调整GRID_NUM值（建议设置为LENGTH/2）
   ```

3. **搜索算法卡死**
   ```
   检查起点和终点是否可达
   尝试增加网格大小或简化迷宫
   使用"显示搜索细节"观察算法过程
   ```

4. **界面显示异常**
   ```
   Windows：确保设置了正确的SDL环境变量
   Linux：可能需要安装tkinter：sudo apt-get install python3-tk
   MacOS：可能需要安装SDL2库
   ```

5. **Dijkstra算法出现闪烁现象**
   ```
   这是已知问题，由于已确定最短距离的点重复入列导致
   可以通过修改算法逻辑解决，但当前版本已进行视觉优化
   ```

## 📚 算法说明

### BFS（广度优先搜索）
- **特点**：逐层扩展，找到最短步数路径
- **适用场景**：所有边权重相等，寻找最短步数
- **时间复杂度**：O(V+E)

### DFS（深度优先搜索）
- **特点**：深入探索分支，快速找到一条路径
- **适用场景**：只需要任意路径，不要求最优
- **时间复杂度**：O(V+E)

### Dijkstra算法
- **特点**：考虑边权重，找到最小代价路径
- **适用场景**：有权图，寻找最小代价路径
- **时间复杂度**：O((V+E)logV)

### A*算法
- **特点**：启发式搜索，更高效找到最优路径
- **适用场景**：有权图，且存在有效启发函数
- **时间复杂度**：取决于启发函数质量

### D-BFS（双向广度优先搜索）
- **特点**：从起点和终点同时进行BFS搜索，在中间相遇
- **适用场景**：大型地图，需要提高搜索效率
- **时间复杂度**：O(V+E)，但通常比BFS更快

### D-DFS（双向深度优先搜索）
- **特点**：从起点和终点同时进行DFS搜索
- **适用场景**：需要快速找到一条路径的大型地图
- **时间复杂度**：O(V+E)

### B*算法
- **特点**：结合最佳优先搜索和A*的改进算法，使用双启发式
- **适用场景**：复杂权重图，需要平衡效率和准确性
- **时间复杂度**：取决于启发函数质量和图结构
